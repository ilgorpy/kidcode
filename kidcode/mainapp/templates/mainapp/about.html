Описание веб-приложения

Введение

Назначение системы

Веб-приложение «KidCode» с функциями учителя предназначена для составления игровых полей в соответствии с выбранным уровнем сложности.

В системе реализовано две роли пользователей: учитель и ученик.

Основные задачи учителя – настройка параметров игрового поля, создание игрового поля по выбранному уровню сложности, проверка кода учеников и выставления им оценок. Ученик может загрузить игровое поле, написать к нему код

В веб-приложении также есть возможность получения справочной информации как о самом приложении, так и о разработчиках.

Условия работы системы

Требования к техническому обеспечению:

1 Требования к техническому обеспечению серверной части:

  • Тип ЭВМ – IBM PC совместимый;
  • Объем ОЗУ – не менее 5 Гб;
  • Объем свободного пространства на внешнем диске – не менее 25 Гб;
  • Наличие подключения к сети Интернет;
  • Манипулятор – мышь;
  • Процессор – AMD Ryzen 3 4300U не менее 2.70 ГГц;
  • Дисплей с разрешением не менее 1024 × 768 пикселей;
  • Операционная система Windows 10 и выше.

2 Требования к техническому обеспечению клиентской части:

  • Тип ЭВМ – IBM PC совместимый;
  • Объем ОЗУ – не менее 5 Гб;
  • Объем свободного пространства на внешнем диске – не менее 21 Гб;
  • Монитор с разрешающей способностью не ниже 1024 × 768;
  • Процессор – AMD Ryzen 3 4300U не менее 2.70 ГГц;
  • Манипулятор – мышь;
  • Операционная система Windows 10 и выше.

Требования к программному обеспечению:

1 Требования к программному обеспечению серверной части:

  • Расширение для C# 2.55.29 и выше;
  • Расширения C# Dev Kit 1.14.14 и выше;
  • .NET Install Tool 2.2.3 и выше;
  • EntityFramework 6 и выше;
  • PostgreSQL 16 и выше.

2 Требования к программному обеспечению клиентской части:

  • Браузер Google Chrome 131.0.6778.205 и выше;
  • Node.js 20.10.0 и выше;
  • React 18.3.1 и выше;

Установка системы

Для запуска веб-приложения необходимо либо склонировать репозиторий с GitHub, либо получить необходимые файлы в виде zip-архива.

1. Перейти в папку kidcode 2. Установить виртуальное окружение python -m venv venv 3. Активировать виртуальное окружение .\venv\Scripts\activate 4. Перейти в следующую папку kidcode и установить пакеты командой pip install -r requirements.txt 5. Запуск проекта python manage.py runserver

Навигация

С помощью кнопок и лого сайта на вверху страницы пользователи могут переходить между различными страницами. На "шапке" соответсвенно расположены лого (для перехлда на авторизацию), навигационная кнопка, ссылка для перехода на страницу конструктора и ссылка для перехода на страницу профиля.

Авторизация

Для входа в аккаунт (учителя или ученика) необходимо выбрать роль и заполнить форму авторизации, представленную ниже, которая расположена на странице авторизации

Пользователь должен заполнить поля "Введите email" и "Введите пароль". Если пользователь вводит запрещенные символы, то под полем будет располагаться соответствующее сообщение (см. рисунок ниже).

После нажатия на кнопку "Войти" осуществляется проверка введённых данных, при успешном завершении которой авторизованный пользователь перенаправляется на страницу профиля.

В случае, если введён несуществующий логин или неправильный пароль, то пользователь увидит сообщение об ошибке.

Работа с системой в режиме учителя

На рисунке снизу представлена страница конструктора.

На данной странице учителю будет предложено два варианта создания задания: автоматически или вручную. При нажатии на кнопку «Автоматически» для генерации игрового поля учитель должен задать уровень сложности задания. Далее ввести текст задания и нажать кнопку «Сгенерировать». Если генерация прошла успешно, требуется нажать на кнопку «Сохранить». В случае, если поля введены некорректно, то учитель должен нажать кнопку «Очистить» для удаления данных в полях.

По нажатию на кнопку «Вручную», учителю требуется ввести параметры игрового поля, выбрать шаблоны, выбрать уровень сложности и ввести текст задания. После заполнения полей, учителю требуется нажать кнопку «Сохранить». Если генерация произошла успешно, то задание сохраняется в БД. В случае, если поля введены некорректно, то учитель должен нажать кнопку «Очистить» для удаления данных в полях.

Работа с журналом

Для работы с журналом учитель должен перейти в профиль, нажав на соответствующую кнопку, а затем выбрать пункт меню "Журнал", где учитель может увидеть все отправленные учениками задания, увидеть дату сдачи и открыть загруженный ими код.

Работа с системой в режиме ученика

Регистрация ученика

На рисунке ниже изображена страница регистрации нового ученика.

В основной части представлена форма для регистрации нового ученика. После ввода почты, имени, пароля и нажатия на кнопку «Зарегистрироваться» осуществляется проверка введённых данных, после которой пользователь проходит регистрацию и перенаправляется на страницу профиля. В случае, если введены некорректные данные, пользователь увидит подсказки под полями (см. рисунок ниже).

Страница задания

На данном рисунке представлена страница задания. В шапке распологаются такие элементы, как всплывающее меню, через которое пользователь может перейти в интересубющее его задание, выбрав главу и уровень в этой главе. Также пользователь может просмотреть справочную информацию о разработчиках, нажав однноименную кнопку. Клик по кнопке "Профиль" позволяет перейти в детали профиля. Если пользователь является учителем - он имеет доступ к конструктору. В боковом меню есть форма, на которой расположена информация о текущем задании, кнопки "Назад" и "Вперёд" для перехода по заданиям в текущей главе, окно для ввода команд и кнопки "Запуск кода", "Очистить" и "Отправить". В основной части страницы расположено игровое поле, с которым взаимодействует пользователь, путем ввода команд.

Страница профиля

На странице детали профиля расположена форма для изменения данных учётной записи. Пользователь может поменять имя и пароль, нажав на кнопку "Сохранить". Наравне с формой в левой части страницы расположено боковое меню, из которого пользователь может перейти в журнал, если он учитель, в успеваемость, если он ученик, а также совершить выход из аккаунта, кликнув по соответсвующей кнопке.
